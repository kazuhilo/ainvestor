# トレーディングサービスを選定する

- Status: Accepted
- Decision-Date: 2026-05-19
- Date: 2026-05-19
- Deciders: kazuhiro, toshiki

## Context

トレーディング機能を構築するにあたり、利用するトレーディングサービス（証券会社・ブローカー API）を選定する必要がある。要件として、開発しやすさ・初期コスト・本番投入前に検証できる仕組み（paper trade）・利用可否（在住国制限）を比較したい。

## Decision

トレーディングサービスとして **Alpaca + Finnhub** を採用する。

- Alpaca: 注文執行・口座管理
- Finnhub: 市況データ・銘柄情報

## Consequences

### Positive

- Alpaca はモダンな API 設計で開発しやすい
- 無料プランから開始でき、初期コストを抑えられる
- paper trade（模擬取引）が使えるため、本番投入前に動作検証ができる

### Negative

- なし

## Alternatives Considered

- IBKR: 使いにくそう（API/SDK の取り回しが煩雑）。よって不採用。
- Tradier: 米国在住者でなければ利用できないため、利用要件を満たせず不採用。
